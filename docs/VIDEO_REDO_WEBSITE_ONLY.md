## Master Prompt

Create a clean product demo video for CAMIKey.com using only footage from the CAMIKey website.

The video must feel like a fresh rebuild from scratch, not a revision. Do not use any old frames, legacy clips, external visuals, abstract graphics, meme inserts, or off-site footage. Every shot should come from the CAMIKey website UI only.

Focus on these website pages and flows:

1. Homepage hero
2. Stream registration on `/start`
3. Advertiser dashboard on `/ads`
4. Stream page on `/<slug>`
5. Overlay page on `/o/[streamId]`

The message is:

"CAMIKey lets live Pump.fun streamers sell Dexscreener chart placements inside an OBS browser source overlay."

Show these beats clearly:

- homepage positioning
- stream registration
- overlay verification
- live advertiser discovery
- buyer mint entry and purchase intent creation
- payment status changing
- chart slot switching inside the overlay

Visual rules:

- use only the website
- no old frames
- no extra instructions on screen
- no lines about deploying GIFs
- no fake popups or fake browser chrome beyond what is needed
- no stock footage
- no device mockups

Editing style:

- crisp cuts
- subtle push-ins
- short pauses on important UI states
- one clear action per shot
- prioritize readability over flashy motion

On-screen text should be minimal and product-forward, such as:

- "Pump.fun chart sponsorships"
- "Live streams only"
- "Verify overlay"
- "Create purchase intent"
- "Chart goes live"

Do not add tutorial-style instructions.
